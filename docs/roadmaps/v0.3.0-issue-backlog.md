# v0.3.0 Issue Backlog

`[Spec]` [v0.3.0 Development Spec](./v0.3.0-development-spec.md)

원칙:

- 각 issue는 하나의 검증 가능한 vertical slice를 가진다.
- 구조 작업은 반드시 benchmark, artifact, 또는 contract 변화와 연결한다.
- `image_embedding`처럼 범위가 커질 수 있는 항목은 spike 또는 stretch로 분리한다.

## Issue 01. Freeze Benchmark Project And Two Target Screens

GitHub issue: [#16](https://github.com/Hanjo92/unity-resource-rag/issues/16)

### Suggested Title

`v0.3.0: freeze benchmark project and two target screens`

### Goal

`v0.3.0` 전체가 같은 입력과 같은 합격 기준을 바라보도록 benchmark baseline을 고정한다.

### Why Now

화면과 기준이 고정되지 않으면 retrieval, repair, gateway 작업이 모두 감으로 흩어진다.

### In Scope

- 기준 Unity 프로젝트 1개 선정
- benchmark 화면 2개 선정
- 각 화면의 reference image, target 설명, expected threshold 정의
- `retrieval-fixture.json`, `screen-fixture.json` 초안 생성

### Out Of Scope

- runner 구현
- quality 개선 자체

### Deliverables

- benchmark project memo
- screen A / screen B definition
- fixture 파일 위치와 naming rule
- acceptance threshold 초안

### Done When

- 팀이 같은 두 화면을 기준으로 이야기할 수 있다.
- fixture schema가 이후 issue에서 재사용 가능하다.

### Dependencies

- 없음

### Suggested Labels

- `v0.3.0`
- `benchmark`
- `planning`

## Issue 02. Extract Gateway Capability Router

GitHub issue: [#17](https://github.com/Hanjo92/unity-resource-rag/issues/17)

### Suggested Title

`v0.3.0: extract gateway capability router from server entrypoint`

### Goal

`pipeline/gateway/server.py`의 단일 hardcoded branching을 capability router 구조로 바꿔 이후 capability 추가를 안전하게 만든다.

### Why Now

`vision_layout_repair_analysis`와 `text_embedding`을 넣기 전에 routing 책임을 정리하지 않으면 server entrypoint가 빠르게 비대해진다.

### In Scope

- capability router interface 정의
- request validation과 capability dispatch 분리
- normalized error mapping 정리
- adapter selection seam 분리

### Out Of Scope

- 새 capability의 실제 모델 품질 개선
- 다중 서버/분산 gateway 설계

### Deliverables

- capability router module
- capability handler registration 방식
- server integration refactor
- regression test for unknown capability / auth error / invalid request

### Done When

- `server.py`는 transport concern 위주로만 남는다.
- 새 capability 추가 시 router registration과 handler 구현만으로 확장 가능하다.

### Dependencies

- Issue 01 권장, 필수는 아님

### Suggested Labels

- `v0.3.0`
- `gateway`
- `architecture`

### Deliverables

- binding report schema update
- binder behavior update
- regression tests for low-confidence branch
- docs note for interpretation

- auto-bind/hold/review 경계가 artifact에 드러난다.
