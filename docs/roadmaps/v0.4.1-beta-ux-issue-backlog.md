# v0.4.1-beta UX Issue Backlog

`[Spec]` [v0.4.1-beta UX Follow-up Spec](./v0.4.1-beta-ux-spec.md)

이 문서는 `0.4.0-beta` real-project validation 이후, Unity-first usability를 높이기 위한 후속 이슈 backlog다.

원칙:

- 각 issue는 사용자가 체감하는 마찰 하나를 줄여야 한다.
- 내부 구조 개선은 반드시 readiness, happy path, case capture 중 하나와 연결한다.
- `unity-mcp`를 대체하려 하지 않고, 그 위에 얹는 애드온 UX를 강화한다.

## Issue 01. Add Readiness Dashboard And Remove Internal MCP Jargon

### Suggested Title

`v0.4.1-beta: add readiness dashboard and user-facing setup language`

### Goal

사용자가 현재 막히는 지점을 `repo`, `python`, `auth`, `Unity MCP`, `build input` 기준으로 이해하게 만든다.

### Why Now

지금은 `doctor` 정보는 많지만 `ui_asset_catalog resource missing` 같은 내부 용어가 그대로 보여서 사용자가 행동으로 옮기기 어렵다.

### In Scope

- Unity window readiness section 추가 또는 개선
- `ready / attention / blocked` 상태 모델 정의
- `doctor` 결과를 user-facing summary로 매핑
- `ui_asset_catalog` resource 같은 내부 용어를 사용자 문장으로 치환

### Out Of Scope

- 실제 bootstrap 자동 수행
- capture/repair flow

### Deliverables

- readiness card UI
- summary formatter
- mapping table for internal diagnostics -> user-facing messages

### Done When

- 처음 보는 사용자가 왜 막혔는지 읽고 바로 다음 액션을 알 수 있다.
- raw MCP jargon 없이도 setup 상태를 설명할 수 있다.

### Dependencies

- 없음

### Suggested Labels

- `v0.4.1-beta`
- `ux`
- `setup`

## Issue 02. Add Python Runtime Bootstrap Assistant

### Suggested Title

`v0.4.1-beta: add Python runtime bootstrap assistant for one-click setup`

### Goal

local checkout 기반 beta에서 가장 흔한 실패인 Python interpreter / dependency mismatch를 창 안에서 복구 가능하게 만든다.

### Why Now

실제 hookup 과정에서 `pydantic` 미설치와 interpreter mismatch가 바로 첫 장애였다.

### In Scope

- interpreter readiness 검사 강화
- requirements install command suggestion 또는 guided action
- repo-local `.venv` 우선 경로 가이드
- bootstrap 결과 report

### Out Of Scope

- 가상환경 관리자 자체 구현
- cross-platform packaging 완성

### Deliverables

- bootstrap assistant UI 또는 action
- improved runtime diagnostics
- docs for safe bootstrap path

### Done When

- 사용자가 terminal 없이도 어떤 interpreter가 필요한지 이해할 수 있다.
- 최소 guided action 한 번으로 sidecar import failure를 복구할 수 있다.

### Dependencies

- Issue 01 권장

### Suggested Labels

- `v0.4.1-beta`
- `ux`
- `runtime`

## Issue 03. Simplify Auth Profiles Around Codex Login Default

### Suggested Title

`v0.4.1-beta: simplify auth profiles around existing Codex login`

### Goal

auth 선택을 `Use existing Codex login`, `API key env`, `Offline local` 정도의 사용자 언어로 고정하고, 내부 preset 개념을 숨긴다.

### Why Now

사용자는 provider preset보다 "지금 로그인 상태를 그대로 쓸 수 있나"를 먼저 생각한다.

### In Scope

- settings UX wording 정리
- auth mode별 readiness message 개선
- Codex auth file detection/report 보강
- API key env 입력 가이드 개선

### Out Of Scope

- provider matrix 확장
- Unity MCP 내부 auth storage 통합

### Deliverables

- simplified auth copy
- auth readiness summaries
- safer defaults for first run

### Done When

- auth 선택지가 provider jargon 없이도 이해된다.
- 기본값으로 `Use existing Codex login`을 쓰는 흐름이 가장 자연스럽다.

### Dependencies

- Issue 01 권장

### Suggested Labels

- `v0.4.1-beta`
- `ux`
- `auth`

## Issue 04. Add Build Phase Progress And Cancellable UX

### Suggested Title

`v0.4.1-beta: add build phase progress and non-blocking run UX`

### Goal

`Start UI Build`가 현재 무엇을 하는지 보여주고, editor freeze 없이 실행되게 만든다.

### Why Now

실사용에서 synchronous wait 때문에 `apply_ui_blueprint` deadlock이 발생했고, 진행 상황도 거의 보이지 않았다.

### In Scope

- build phases 정의
  - readiness
  - catalog reuse/index
  - draft/reference route selection
  - blueprint generation
  - validate/apply
- running state UI
- cancel 또는 abort semantics 설계
- improved success/failure summaries

### Out Of Scope

- capture/repair 실행

### Deliverables

- progress model
- window progress rendering
- cancellation or safe ignore strategy

### Done When

- 사용자가 build가 멈췄는지, 진행 중인지, 어느 단계인지 알 수 있다.
- Unity editor가 blocking wait로 멈추지 않는다.

### Dependencies

- 없음

### Suggested Labels

- `v0.4.1-beta`
- `ux`
- `build-flow`

## Issue 05. Generalize Catalog Draft Modes Beyond Popup

### Suggested Title

`v0.4.1-beta: add template-aware catalog draft modes for popup, HUD, and list UIs`

### Goal

현재 popup 중심인 catalog-first draft를 HUD, inventory/list 패턴까지 작게 일반화한다.

### Why Now

실프로젝트 품질 보강은 다양한 화면 패턴에서 case를 모아야 하고, 현재 draft 진입점은 popup에 치우쳐 있다.

### In Scope

- template mode selector
- popup / hud / list draft skeleton
- mode별 search query defaults
- mode별 user copy

### Out Of Scope

- full auto-layout generation
- complex nested scroll/list productionization

### Deliverables

- template mode input
- mode-aware draft builder
- sample outputs for 2~3 patterns

### Done When

- reference 없이 시작해도 popup 외 화면에 대한 초안이 가능하다.
- mode를 바꾸면 asset query와 blueprint skeleton이 달라진다.

### Dependencies

- Issue 04 권장

### Suggested Labels

- `v0.4.1-beta`
- `ux`
- `generation`

## Issue 07. Add Last-Run Artifact Browser And Reopen Actions

### Suggested Title

`v0.4.1-beta: add last-run artifact browser for blueprint, report, and handoff outputs`

### In Scope

- Finder/open folder actions
- blueprint/search report/handoff quick links
- last selected canvas/object summary

### Out Of Scope

- full asset browser replacement

### Deliverables

- artifact panel
- quick-open actions
- result memory model

### Done When

- 최근 실행 결과를 창 안에서 다시 열 수 있다.
- artifact path를 수동으로 복사하지 않아도 된다.

### Dependencies

- Issue 04 권장

### Suggested Labels

## Issue 09. Decide Packaged Sidecar Distribution Strategy

### Suggested Title

`v0.4.1-beta: decide packaged sidecar distribution strategy for non-dev users`

### Goal

`file:` local checkout과 Python repo root 의존을 줄이기 위한 distribution 방향을 결정한다.

### Why Now

지금 one-click UX는 개발자 머신에서는 괜찮지만, 비개발 사용자에게는 여전히 설치 장벽이 크다.

### In Scope

- spike: sidecar bundle options 조사
- repo checkout dependency 정리
- `.venv` / embedded runtime / installer 전략 비교
- keep/defer recommendation

### Out Of Scope

- full production packaging 완성

### Deliverables

- decision memo or ADR
- cost/risk comparison
- recommended next slice

### Done When

- 다음 major usability step이 packaged distribution인지, guided local checkout 유지인지 결정된다.

### Dependencies

- 없음

### Suggested Labels

- `v0.4.1-beta`
- `spike`
- `distribution`
