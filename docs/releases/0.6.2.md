# 0.6.2

`unity-resource-rag` 0.6.2 is a patch release focused on hardening release packaging, runtime detection, and recovery paths around the Unity sidecar.

## Highlights

- preserves nested `include_image` capture requests instead of dropping them at the local runner boundary
- rejects unexpected batched embedding payloads explicitly and retries Unity MCP calls once after session reinitialization
- aligns the project on Python 3.11+ runtime detection and publishes portable sidecar bundles as GitHub release assets
- exposes the blueprint sample through UPM and resolves `Samples~/...` paths correctly when the package is installed

## Included Changes

### Runtime hardening

- `capture_result` now keeps caller-provided nested `include_image` values intact
- retrieval embedding normalization now rejects multi-item gateway batches with a clear error instead of silently picking one shape
- Unity MCP HTTP calls recover once from session-related status codes by reinitializing the session and retrying the original request
- benchmark fixture loading now rejects invalid `repeatCount`, `interactionLevel`, and `bindingPolicy` values earlier

### Packaging and release discipline

- added CI coverage for the Python sidecar, compile checks, and a bundle smoke build in GitHub Actions
- added a release workflow that publishes `unity-resource-rag-sidecar-<version>.zip` as a GitHub release asset
- documented Python 3.11+ requirements more consistently across the root README, Unity package README, contributor guide, and Unity editor messaging
- added issue templates for bug reports, docs issues, and installed package or release artifact regressions

### Unity package distribution

- registered `Samples~/Blueprints` in the UPM manifest so the package ships a visible blueprint starting point
- documented which sample references are project-specific and need local adaptation
- resolved `Samples~/...` blueprint paths relative to the installed UPM package and replaced fragile package-doc relative links with stable GitHub URLs

## Validation

- `python3.12 -m unittest discover -s tests -v`
- `python3.12 -m compileall pipeline`
- `python3.12 scripts/build_sidecar_bundle.py --output-dir <tmp>`
