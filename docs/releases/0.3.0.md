# 0.3.0

Gateway-first benchmark and repair infrastructure release for `unity-resource-rag`.

## Highlights

- Expanded the gateway from a single extraction slice into a multi-capability runtime with `vision_layout_extraction`, `vision_layout_repair_analysis`, `text_embedding`, and preview `image_embedding`.
- Added retrieval quality infrastructure: low-confidence binding states, candidate preservation, and sparse embedding bridge reuse.
- Added structured repair patch candidate generation so verification output can be handed off as bounded node-level actions.
- Added benchmark runners, sample fixtures, and a checked-in baseline gate report for the two-screen `v0.3.0` reference suite.
- Published team troubleshooting guidance and an ADR that records the preview-only scope for `image_embedding`.

## Included In This Release

### Gateway And Provider Runtime

- capability router extraction from the gateway server entrypoint
- `vision_layout_repair_analysis` capability
- `text_embedding` capability for retrieval experiments
- preview `image_embedding` capability that emits sparse visual-token outputs

### Retrieval And Repair Quality

- low-confidence binding flow with `auto_bind`, `hold`, and `review_needed`
- candidate-preserving binding reports
- generic sparse embedding bridge scoring
- structured repair patch candidate builder

### Benchmarks

- `v0.3.0-reference-suite` manifest, fixtures, and sample inputs
- `run_retrieval_benchmark.py`
- `run_screen_benchmark.py`
- `publish_benchmark_gate_report.py`
- checked-in baseline artifacts under `examples/benchmarks/v0.3.0-reference-suite/baseline-artifacts/`

### Docs

- gateway auth and benchmark troubleshooting guide
- roadmap/backlog updates for completed `v0.3.0` scope
- image embedding preview scope ADR

## Notes

- The checked-in benchmark gate is based on sample inputs and is intended to be a deterministic engineering baseline.
- For stricter production sign-off, rerun the same benchmark gate with real project captures before shipping a release candidate.
