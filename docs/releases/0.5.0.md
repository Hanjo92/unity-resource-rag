# 0.5.0

`unity-resource-rag`의 첫 stable Unity-first usability release.

## Highlights

- Unity 안에서 `Quick Setup -> Readiness -> Start UI Build -> Capture & Repair -> Case Export`까지 한 창에서 이어갈 수 있다.
- `Readiness Dashboard`가 `repo / python / provider login / Unity Editor connection / build input` 상태를 사용자 언어로 요약한다.
- `Bootstrap Python Runtime`으로 repo-local `.venv`와 requirements를 창 안에서 준비할 수 있다.
- `Start UI Build`는 이제 readiness 확인과 build/apply를 비동기로 나눠 실행해 editor freeze 없이 진행 상황을 보여준다.
- build 후에는 `Capture Result`와 `Run Repair Handoff`로 follow-up verification을 바로 이어갈 수 있다.
- `Export Case Report`는 `Library/ResourceRag/Cases/` 아래에 md/json 품질 리포트를 남겨 real-project follow-up을 수집하기 쉽게 만든다.

## Why This Is Stable

- 실제 Unity 프로젝트에서 `catalog -> blueprint -> Unity object generation` 흐름을 검증한 뒤, first-run UX에서 막히던 setup/runtime/timeout/deadlock 문제를 창 안에서 복구 가능한 수준까지 줄였다.
- beta에서 드러난 핵심 마찰이던 Python interpreter mismatch, readiness 불투명성, blocking build, capture/repair 단절을 이번 릴리즈에서 직접 다뤘다.

## Known Limitations

- Unity window 기반 one-click flow는 여전히 full local checkout 경로가 필요하다.
- `Run Repair Handoff`는 reference image가 있을 때만 동작한다.
- 더 넓은 화면 유형 일반화와 artifact browser, packaged sidecar distribution은 후속 릴리즈 범위다.
