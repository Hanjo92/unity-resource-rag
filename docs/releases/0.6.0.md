# 0.6.0

`unity-resource-rag` 0.6.0 focuses on the next usability step after the Unity-first 0.5.0 release: making the Unity window easier to understand, widening catalog-first draft coverage, and reducing the checkout-only assumption around the Python sidecar.

## Highlights

- simplified the Unity window sign-in UX around three clear paths:
  - `Use my Codex sign-in (Recommended)`
  - `Use an API key from my environment`
  - `Stay offline with local fallback`
- expanded catalog-first draft mode beyond popup-only flow so Unity users can choose `Popup / Modal`, `HUD / Top Bar`, or `List / Inventory`
- added `Last Run Artifacts` to the Unity window so blueprint, search report, handoff, capture, repair, and case export outputs can be reopened without digging through logs
- standardized MCP tool descriptions, provider setup summaries, and doctor diagnostics in English to avoid `?` rendering in MCP clients
- accepted `portable sidecar bundle + sidecar-local .venv bootstrap` as the recommended non-dev distribution direction and added a sidecar bundle build script

## Included Changes

### Unity window UX

- auth profile wording now stays on user-facing language instead of low-level provider preset jargon
- Codex auth file overrides are hidden under the advanced section by default
- catalog-first draft templates now carry through Unity settings, local runner payloads, and report summaries
- recent build, capture, repair, and case export artifacts can be reopened from the same window

### Sidecar and MCP surfaces

- provider inspection and doctor outputs are standardized in English
- hidden version metadata is aligned to `0.6.0` across the package, MCP server, gateway server, and Unity HTTP helper

### Distribution decision

- new ADR: [packaged-sidecar-distribution-strategy.md](../decisions/packaged-sidecar-distribution-strategy.md)
- new bundle builder:

```bash
python3 scripts/build_sidecar_bundle.py --output-dir dist
```

This script produces a portable sidecar runtime folder that can be used as the Unity window's `Sidecar Runtime Root`.

## Validation

- `python3 -m unittest discover -s tests -v`
- `git diff --check`
- `python3 scripts/build_sidecar_bundle.py --output-dir <temp-dir>`

## Notes

- contributor setup with a full local checkout remains supported
- non-dev packaging is now aligned around a portable sidecar bundle instead of an embedded Python runtime
- the long-lived untracked example draft blueprint remains outside the release payload
